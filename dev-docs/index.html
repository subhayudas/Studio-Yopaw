The big picture in 4 steps
1

You write code in TypeScript and React (frontend) + TypeScript (backend)

TypeScript is JavaScript with types — like Python with type hints but enforced at compile time. React is a JavaScript library for building UIs. Your API files in api/ look like Python Flask handlers but written in TypeScript.

2

Vite compiles and bundles the frontend

Vite turns your React+TypeScript code into plain HTML+CSS+JS that any browser can read. This is like compiling a C program — the output is simpler than the source. The output goes into a dist/ folder.

3

Vercel deploys two things at once

It serves the dist/ folder as static files (like Nginx serving HTML), AND it takes every file in api/*.ts and turns each one into a separate serverless function — an HTTP endpoint that runs on demand.

4

Users hit one domain, Vercel routes everything

www.yopaw.ca/ serves the React app. www.yopaw.ca/api/booking runs your api/booking.ts function. Same domain, completely different systems underneath.

Why is this interesting for your portfolio site?

Vercel lets you build a static frontend (just HTML/CSS/JS — no server required) AND serverless backend functions, all from one repo, all on the same free tier. For a portfolio site you probably only need the static side — but knowing how the API layer works means you can add a contact form, analytics endpoint, or any dynamic feature later without spinning up a separate server.
