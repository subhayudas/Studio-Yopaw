Resources to go deeper

Everything you need to fully understand and independently replicate this architecture — ordered from "start here" to "advanced". Tailored for a Python developer.

On this page

How to use this list Don't try to learn everything at once. Follow the phases in order. For your portfolio site, Phases 1 and 3 are the most directly relevant. Phase 2 will deepen your understanding. Phase 4 is only needed if you want to build something as complex as Studio Yopaw.

Phase 1 — JavaScript / TypeScript for Python developers

Start here
📄
JavaScript for Python developers (MDN)
Free Article MDN (Mozilla Developer Network) is the gold-standard reference for everything web. Read "JavaScript — Dynamic client-side scripting" — it's the most accurate reference that exists, written like Python docs. Search "MDN JavaScript guide" and start with the basics section.
🎥
TypeScript for Python Developers — Fireship (YouTube)
Free Video Search "Fireship TypeScript in 100 seconds" on YouTube. Fireship makes extremely dense, accurate 100-second explainers. Then watch "TypeScript for JavaScript Developers" for the full version (~10 min). Fireship's style is perfect — no fluff, technically sharp.
📄
TypeScript official handbook — "TypeScript for Java/OOP programmers"
Free Docs The TypeScript docs have a specific page called "TypeScript for Java/OOP Programmers" — this applies equally well to Python devs. Search "typescriptlang.org for java programmers". It directly addresses the conceptual differences that trip up people from statically-typed or scripting backgrounds.
🎮
JavaScript30 by Wes Bos
Free Interactive 30 small projects in 30 days, in plain JavaScript (no frameworks). Visit javascript30.com. This is the fastest way to get fluent with JavaScript by doing real things rather than reading theory. You'll learn DOM manipulation, events, fetch, async/await — all the concepts this project uses.

Phase 2 — The JavaScript toolchain (Vite, npm, React)

Learn the tools
📄
Vite official documentation — "Why Vite"
Free Docs Go to vitejs.dev and read the "Why Vite" page and "Getting Started". The docs are excellent and short. Pay special attention to the "Features" section which explains ES modules, HMR (Hot Module Replacement), and why Vite is much faster than older tools like Webpack. This is exactly what powers this project.
🎥
React in 100 seconds — Fireship (YouTube)
Free Video Search "Fireship React in 100 seconds" on YouTube. Then watch "React Hook Tutorial" (~10 min) to understand useState and useEffect — the two hooks used throughout this project. Understanding hooks is essential before reading any of the src/ code in Studio Yopaw.
📄
React official documentation — react.dev
Free Interactive The new React docs (react.dev) are outstanding — they have interactive examples and are written for people who know programming but not React. Work through the "Learn" section: "Describing the UI", "Adding Interactivity", "Managing State". Stop there for now — that covers everything used in this project.
📄
Tailwind CSS documentation — tailwindcss.com
Free Docs Tailwind is the CSS framework used for styling. You don't need to learn it deeply for the architecture docs, but understanding that className="flex items-center gap-4 text-gray-500" is Tailwind utility classes (not custom CSS) will help you read the component code. The "Core Concepts" section takes 30 minutes.

Phase 3 — Vercel and serverless

The deployment layer
📄
Vercel documentation — "Serverless Functions"
Free Docs Go to vercel.com/docs and find "Functions" → "Serverless Functions". Read the overview, the Node.js runtime section, and the "API Routes" section. This will show you exactly how api/booking.ts in this project gets turned into a real HTTP endpoint. Also read "Environment Variables" — critical for understanding secrets management.
🎥
Vercel in 100 seconds — Fireship (YouTube)
Free Video Search "Fireship Vercel in 100 seconds" on YouTube. Perfect 100-second overview of what Vercel does and why people use it. Great starting point before reading the docs.
📄
Vercel documentation — "Deploy a static site"
Free Docs Directly relevant for your portfolio. Search "Vercel deploy static site" or go to vercel.com/docs/frameworks/vite for the Vite-specific guide. You'll see how simple deployment actually is — push to GitHub, connect the repo, done. The CDN, HTTPS, and custom domain all come for free.
📄
Serverless Functions vs Traditional Servers (blog post)
Free Article Search "serverless vs traditional server comparison" on the Vercel blog (vercel.com/blog) or Cloudflare's blog. The Cloudflare blog (blog.cloudflare.com) in particular has excellent deep-dives on serverless architecture written for a technical audience. Search "what is serverless computing cloudflare".

Phase 4 — Advanced (understanding this specific project)

Deep dive
📄
Vite — "Plugin API" documentation
Free Docs Go to vitejs.dev/guide/api-plugin.html. This explains how devApiPlugin() works at a deep level — the configureServer hook, the Connect middleware interface, and how Vite's plugin system is designed. You'll need the basics of JavaScript first (Phase 1) before this will make sense.
📄
Node.js — "HTTP module" documentation
Free Docs Node.js is the JavaScript runtime that runs your api/*.ts functions. Search "Node.js HTTP docs" at nodejs.org/docs. The req and res objects in Vite's middleware and in the Vercel functions are Node.js IncomingMessage and ServerResponse objects — understanding these unlocks the server code completely.
📄
Connect — middleware library (Node.js)
Free Docs Search "connect npm middleware" on GitHub (github.com/senchalabs/connect). Connect is the minimal middleware framework Vite's dev server is built on. Reading its README (10 minutes) will make server.middlewares.use() completely clear. Express.js is built on top of Connect too — learning one teaches both.
📄
React Custom Hooks — "Building Your Own Hooks" (react.dev)
Free Docs This project uses three custom hooks: useSquareAvailability, useBreedSchedule, useSquareCard. Understanding how custom hooks work will let you read these files clearly. Search "react.dev building your own hooks" — it's a page in the official docs with interactive examples.

Quick glossary — every term used in these docs

All technical terms used across these 5 pages, in one place:

SPA (Single-Page Application)
A website where all navigation happens in the browser via JavaScript — no new HTML file is fetched for each "page". React apps are typically SPAs.
Vite
A build tool and dev server for JavaScript projects. Compiles TypeScript and React on the fly. Much faster than older tools like Webpack.
TypeScript
JavaScript with optional static types. Like Python with type hints, but the types are checked at compile time and prevent the code from running if there are errors.
JSX / TSX
A syntax extension for JavaScript/TypeScript that lets you write HTML-like code inside JavaScript functions. .tsx is TypeScript + JSX. Compiled to plain JavaScript by Vite.
Serverless function
A function that runs on a server only when called, then shuts down. You pay only for execution time. No persistent process — opposite of a traditional always-running server.
Lambda
Amazon's name for serverless functions (AWS Lambda). Vercel Functions are built on top of AWS Lambda under the hood. The terms are often used interchangeably in the industry.
CDN (Content Delivery Network)
A network of servers distributed worldwide that cache and serve your static files. Vercel deploys your dist/ folder to a CDN so users get files from a nearby server.
HMR (Hot Module Replacement)
Vite's feature that updates your browser instantly when you save a file — without a full page reload and without losing the current UI state.
Middleware
A function that runs between an incoming HTTP request and the final handler. Can modify the request, respond directly, or pass it along. Used extensively in Node.js web frameworks.
Node.js
A JavaScript runtime that runs JavaScript on the server (not in a browser). Like CPython for Python — it's the engine that executes your api/*.ts functions on Vercel.
npm
Node Package Manager. The Python equivalent of pip. npm install installs packages from a registry. package.json is the equivalent of requirements.txt.
Cold start
The extra latency (200ms–1s) when a serverless function is called for the first time after being idle. The platform needs to create a new container. Subsequent calls are faster ("warm").
Env var (environment variable)
A key-value pair set outside your code (in the OS or deployment platform) to store secrets like API keys. In Python: os.environ.get('KEY'). In Node.js: process.env.KEY.
Cache busting
Adding a hash (fingerprint) to a filename — e.g., index-a3f2b1.js. If the file changes, the hash changes, so browsers download the new version instead of using their cached copy.
Vite plugin
An object with specific properties that Vite calls at different points in its lifecycle. The devApiPlugin() in this project is a Vite plugin that adds custom middleware to the dev server.
Webhook
A reverse API call. Instead of your code calling Square's API to ask "did anything happen?", Square calls YOUR endpoint (/api/square-webhook) to tell you when an event occurs.
Final advice The best way to learn all of this is to build something. Take the concepts from these docs and deploy a simple portfolio site on Vercel this week. You'll hit real problems, Google the errors, and that's how it sticks. The resources above will fill in the gaps as you encounter them — not all at once before you start.
