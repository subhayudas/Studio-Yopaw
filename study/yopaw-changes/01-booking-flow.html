Flow Diagram: Yin (Regular Class)

Before

chooseClass → public · date → public · contact → public · payment → publicSuccess (regular)

After (new mat step inserted)

chooseClass → public · mat ★ → public · date → public · contact → public · payment → publicSuccess (regular)

The Gentle (Private Event) and Corporate flows are unchanged — they go directly to the people step.

Flow Type Union — What Changed

In src/App.tsx, the Flow discriminated union had the public step literal updated:

- | { kind: 'public'; step: 'people' | 'date' | 'contact' | 'payment'; yoga: YogaStyle }
+ | { kind: 'public'; step: 'mat' | 'people' | 'date' | 'contact' | 'payment'; yoga: YogaStyle }

YogaStyle is 'yin' | 'gentle'. The mat step is only ever entered from the yin path.

New State: needsMatRental

const [needsMatRental, setNeedsMatRental] = useState(false)

A boolean that lives in PricingSection. It is:

  • Set to true when the user picks "No, I will rent one on-site"
  • Set to false when the user picks "Yes, I'll bring my own" or on resetFlow()
  • Read in the payment step to add $5 (500 cents) to baseAmountCents
  • Read in the payment summary to show a separate "Mat rental" line item

The pickMat(renting) Handler

const pickMat = (renting: boolean) => {
  setNeedsMatRental(renting)
  requestScrollPricingCardAfterAdvance()
  setFlow(prev =>
    prev.kind === 'public' && prev.step === 'mat'
      ? { ...prev, step: 'date' }
      : prev
  )
  setSelectedSessionIso(null)
  setPendingSessionIso(null)
  setSelectedTimeSlotId(null)
}
Why reset the date state here? The user hasn't seen the date step yet when they pick the mat option, but if they somehow came back to this step (via "Back") after already picking a date, those stale values would cause the next step to skip the date picker. Clearing them guarantees a clean slate.

Entry Point: pickClass(id) Change

When the user clicks a class card, pickClass decides the first step:

- const firstStep = id === 'gentle' ? 'people' : 'date'
+ const firstStep = id === 'gentle' ? 'people' : 'mat'

So selecting "Regular Class" (yin) now lands on mat instead of jumping straight to date.

Back Navigation from mat Step

The goBack() function handles the mat step:

if (flow.kind === 'public') {
  if (flow.step === 'mat') {
    setFlow({ kind: 'chooseClass' })
    return
  }
  // ...
  if (flow.step === 'date') {
    if (flow.yoga === 'gentle') {
      setFlow({ kind: 'public', step: 'people', yoga: flow.yoga })
    } else {
      // ← previously went to chooseClass; now goes back to mat
      setFlow({ kind: 'public', step: 'mat', yoga: flow.yoga })
    }
    // ...
  }
}

And the showBack flag now includes the mat step:

+ if (flow.kind === 'public' && flow.step === 'mat') showBack = true
else if (flow.kind === 'public' && flow.step === 'people') showBack = true

Mat Rental in the Payment Total

Amount sent to the API

- baseAmountCents = serviceInfo.baseAmountCents
+ baseAmountCents = serviceInfo.baseAmountCents + (needsMatRental ? 500 : 0)

So if the user chose to rent a mat, 500 cents ($5 CAD) is added to the order before taxes are applied. The GST and QST are then computed on top of the new total, so the mat rental is also taxed.

Payment summary UI

In the payment step, the price breakdown was updated to show the mat rental as a separate line:

const matRentalCents = flow.yoga === 'yin' && needsMatRental ? 500 : 0
const baseCents = svc.baseAmountCents * groupSize + matRentalCents

// Base session line shows WITHOUT mat rental
<span>{fmtCAD(baseCents - matRentalCents, lang)}</span>

// Conditional mat rental line
{matRentalCents > 0 && (
  <div className="pricing-payment-summary-row">
    <span>{lang === 'fr' ? 'Location de tapis' : 'Mat rental'}</span>
    <span>{fmtCAD(matRentalCents, lang)}</span>
  </div>
)}

New groupSize Field Sent to Booking API

The booking request body now includes a groupSize field used by Zapier webhooks:

body: JSON.stringify({
  // ... existing fields
  groupSize:
    flow.kind === 'corporate' || (flow.kind === 'public' && flow.yoga === 'gentle')
      ? privateGroupCount
      : '1',
})

For regular yin sessions the server receives '1'. For private/corporate it receives the selected group size (e.g., '8').
