Testimonials Section — Removed

The TestimonialsSection component, along with its supporting components TestimonialCarouselArrow and StarIcon, were entirely removed from src/App.tsx.

What was removed

  • StarIcon() — SVG star used in the 5-star rating display (18 lines of JSX)
  • TestimonialCarouselArrow({ dir }) — left/right arrow SVG (18 lines)
  • TestimonialsSection() — the full carousel component: auto-advance timer, prev/next buttons, dot navigation, testimonial cards with quotes and author info (~107 lines)

Where it was removed from MarketingSite

<PricingSection />
- <TestimonialsSection />
<GallerySection />
The i18n strings for testimonials (testimonialCards, testimonialsBadge, etc.) are still in siteStrings.ts and the SiteStrings interface — they were not cleaned up. They are just unused strings for now.

Pricing Card Header — Simplified

The pricing card previously showed a different header depending on which step the user was on. The logic was refactored.

Before — two separate display conditions

const showFullPricingCardHeader = flow.kind === 'chooseClass'

{showFullPricingCardHeader ? (
  <>
    <div className="pricing-amount">
      <span className="price-value">{s.pricingAmount}</span>
      <span className="price-suffix">{s.pricingPlusTaxes}</span>
    </div>
    <div className="price-type">{s.pricingDropInRow}</div>
    <ul className="price-features">
      {[s.pricingFeat1, ..., s.pricingFeat5].filter(...).map(...) }
    </ul>
    <hr className="pricing-card-divider" />
  </>
) : (
  !isBookingSuccessScreen &&
    <p className="pricing-header-summary">{s.pricingHeaderSummary}</p>
)}

After — single condition, always show summary during booking steps

const showPricingStepSummary = !isBookingSuccessScreen && flow.kind !== 'chooseClass'

{showPricingStepSummary && (
  <>
    <p className="pricing-header-summary">{s.pricingHeaderSummary}</p>
    <ul className="price-features">
      {[s.pricingFeat1, s.pricingFeat2]  // ← only 2 features, not 5
        .filter(line => line.trim().length > 0)
        .map((line, i) => <li key={i}>{line}</li>)}
    </ul>
    <hr className="pricing-card-divider" />
  </>
)}
Flow stateBeforeAfter
chooseClass Full header: price, drop-in label, 5 features, divider Nothing (no header shown during class selection)
Any booking step (mat, date, contact, payment) Summary text only (no features list) Summary text + 2-item features list + divider
Success screens Nothing Nothing
Why only pricingFeat1 and pricingFeat2? pricingFeat3, pricingFeat4, and pricingFeat5 are currently empty strings ('') in siteStrings.ts. The filter .filter(line => line.trim().length > 0) would remove them anyway, so switching to explicitly only using the first two is just cleaner.
