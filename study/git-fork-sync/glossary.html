Core terms

Commit

A snapshot of your project with a unique ID (hash) like f3bcadb83c.... Commits form a history chain.

Branch

A named pointer to a commit. Example: main points to a specific commit. When new commits are added, the branch pointer moves forward.

HEAD

The commit your current checkout points to. Usually it’s “the tip” of your current branch. You can inspect it with git rev-parse HEAD.

Remote

A saved nickname for a Git server URL. Example: origin and upstream. You can list them with git remote -v.

Remote-tracking branch

A local reference to “what the remote branch looked like the last time you fetched”. Examples: origin/main, upstream/main.

Operations

Fetch

Downloads updates from a remote into remote-tracking branches, but does not modify your working files. Example: git fetch upstream.

Merge

Combines two histories. Example: git merge upstream/main brings upstream changes into your local branch.

Merge conflict

Happens when Git can’t automatically combine changes to the same lines/files. Git marks affected files as “unmerged” until you resolve them.

Reset (hard)

Moves your branch pointer and rewrites your working directory to match a target commit. Example: git reset --hard origin/main makes your local branch match the fork’s branch.

Hard reset warning

It can delete uncommitted work. Use backups/branches if you might need the work later.

Prune

When fetching, --prune cleans remote-tracking branches that were deleted on the server.

Fork workflow vocabulary

Origin

Usually your fork’s remote. You push your work here: git push origin main.

Upstream

Usually the original project’s remote. You fetch from here to get new changes: git fetch upstream.
