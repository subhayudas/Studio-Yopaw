One-time setup (recommended)

Add the upstream remote (original repo) and disable pushing to it: 

git remote add upstream https://github.com/<UPSTREAM_OWNER>/<REPO>.git
git remote set-url --push upstream DISABLE
Why disable pushing?

A fork workflow usually means: push to your fork (origin), and pull/fetch from upstream (upstream). Disabling upstream pushes prevents mistakes.

Regular sync (safe baseline)

Fetch new commits from upstream, then integrate them into your local main: 

git fetch upstream --prune
git checkout main
git merge upstream/main

--prune cleans up remote-tracking branches that were deleted on the server.

Merge vs rebase

merge creates a merge commit when needed. rebase rewrites your local commit history on top of upstream. Either can work; if you’re not sure, start with merge.

Check if your fork is behind upstream

This compares two branches and tells you how many commits each side has that the other doesn’t: 

git rev-list --left-right --count origin/main...upstream/main
  • 0 0 means they’re identical
  • 0 N means your fork is behind by N commits
  • N 0 means your fork has commits upstream doesn’t (rare in a strict fork workflow)

Push the synced result to your fork (origin only)

After merging upstream into local main: 

git push origin main
This does NOT push to upstream

Only pushes to the remote named origin. With upstream pushes disabled, you’re doubly protected.
