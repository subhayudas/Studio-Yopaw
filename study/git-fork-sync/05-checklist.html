Situation 1 — “I just want my local repo to match my fork (discard local changes)”

Warning

This can delete local uncommitted changes. If you might need them later, back up first. 

git merge --abort  2> NUL
git fetch origin
git checkout main
git reset --hard origin/main

Note: 2> NUL is a Windows/cmd-style redirect. In PowerShell, you can omit it or use 2>$null.

Situation 2 — “I want to sync my fork with upstream (typical fork workflow)”

git remote -v
git fetch upstream --prune
git checkout main
git merge upstream/main
git push origin main
Tip

Run git status -sb before pushing to ensure you’re on the right branch.

Situation 3 — “Prove origin/main and upstream/main are identical”

git fetch origin --prune
git fetch upstream --prune
git rev-list --left-right --count origin/main...upstream/main
git rev-parse origin/main
git rev-parse upstream/main

Situation 4 — “Make upstream impossible to push to”

git remote set-url --push upstream DISABLE
git remote -v
