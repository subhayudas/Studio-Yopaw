Goal

You said: “discard the changes I have done, and just make it so I have commit f3bcadb in the forked repo and on my machine.”

Outcome: your local branch main and your fork’s remote branch origin/main both point to f3bcadb (same commit hash).

Final state check 
git rev-parse HEAD
git rev-parse origin/main
git show -s --oneline --decorate HEAD

If the first two hashes match, your local branch and the fork’s branch are aligned.

What we changed (conceptually)

Safety

We prevented “oops push to upstream”

We added an upstream remote (the original repo) and disabled its push URL so it can’t be pushed to from this clone.

Cleanup

We exited an unfinished merge

The repo was mid-merge with conflicts. We backed up files and ran git merge --abort so Git returned to a consistent state.

Sync

We aligned your local branch to the fork

We moved local main to match origin/main using git reset --hard origin/main.
